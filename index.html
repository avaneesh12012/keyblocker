Devlog

Development Log.

A running log of what's been built, decided, and figured out.

Mar 2025
Project started — why C++?

KeyBlocker started as a personal problem — the Windows key kept interrupting gaming sessions. Decided to build a proper solution in C++ because it gives direct access to the Windows API with near-zero overhead. Python would work but use 10x more RAM. C# would work but add a .NET runtime. C++ with the Windows API directly is the right tool.

Mar 2025
Low-level keyboard hook working

Got SetWindowsHookEx working with WH_KEYBOARD_LL. The hook intercepts keystrokes system-wide before they reach any application. Blocking the Windows key in fullscreen games works reliably. RAM usage measured at ~1.2MB. First milestone complete.

Coming soon
3D UI design started

Setting up OpenGL with GLFW and Dear ImGui for the 3D keyboard interface. Each key will be a box geometry with a top face and side faces drawn as flat polygons — achieving a 3D look with clean geometric shapes. Click detection uses per-key axis-aligned bounding boxes projected into screen space.

Privacy Policy

Your Privacy.

KeyBlocker collects nothing. Here's exactly what that means.

No Keystroke Logging

KeyBlocker intercepts keystrokes to block them — it never records, stores, or transmits any keystroke data. The blocked key list exists only in RAM and is lost when the app closes.

No Network Requests

KeyBlocker makes zero network requests during normal operation. It works fully offline. The only time it contacts the internet is if you explicitly check for updates in the settings.

No Data Collection

No analytics, no crash reporting, no telemetry, no account required. The app has no idea who you are and no mechanism to find out. Source code is open for anyone to verify.

Alongside KeyBlocker, he is developing an original game in CryEngine 5.7 LTS with fully custom 3D assets modelled and rigged in Blender 5.0. Every mesh, texture, and animation in the game is made by hand.

He chose C++ for KeyBlocker because it gives direct access to the Windows API with minimal overhead — the same reason game engines like CryEngine and Unreal are built in C++. The goal was always a tool that runs invisibly and consumes near-zero resources.

KeyBlocker started as a personal problem: the Windows key kept interrupting gaming sessions. Instead of using an existing tool, he decided to build his own — and turned it into a full app with a 3D interface, keyboard shortcuts, gaming mode, and a multi-platform release plan.

Press Description

KeyBlocker is a free, open-source keyboard key-blocking utility for Windows 10 and 11. Built in C++ with a 3D OpenGL interface powered by Dear ImGui, it lets users block any keyboard key with a single click on an interactive 3D mechanical keyboard model.

The app features a Gaming Mode that instantly blocks all keys known to interrupt gameplay — including the Windows key — and runs in the system tray at near-zero RAM usage. Developed by a 13-year-old independent developer from India, available free on GitHub and itch.io.
